# IT Аутсорс

Инструмент учёта клиентов, выездов и расчёта стоимости работ для выездного IT-специалиста. Работает как веб-приложение в браузере и как нативное Android-приложение через Capacitor.

## Возможности

**Счёт**
- Кнопка «📄 Счёт» в карточке клиента при активном фильтре «Не оплачено»
- Список работ из всех неоплаченных выездов с чекбоксами (все включены по умолчанию)
- Предпросмотр текста счёта в реальном времени при снятии/установке галочек
- Экспорт через системный диалог «Поделиться» (APK) или `navigator.share` / буфер обмена (браузер)

**Прайс-лист**
- Справочник работ с ценой за единицу
- Поиск по названию
- Алфавитная сортировка

**Клиенты**
- Карточка клиента: название, адрес, множитель наценки, заметки
- Несколько контактных лиц на клиента — имя, должность, телефон
- Поиск по названию, адресу, контактам, должностям и телефонам
- Алфавитная сортировка
- Фильтры: В работе / Не оплачено / Сделано (с счётчиком)

**Выезды**
- История выездов по каждому клиенту, новые сверху
- Статусы: В работе (янтарный), Не оплачено (красный), Оплачено
- Причина вызова — свободный текст или вставка из прайса
- Список работ из прайса с указанием количества и множителя
- Множитель наследуется от клиента, редактируется для каждой позиции отдельно
- Ручной ввод итоговой суммы с янтарной индикацией
- Произвольные заметки к выезду

**Интерфейс**
- Светлая и тёмная тема (запоминается)
- Встроенная справка с историей версий
- Мобильный дизайн — крупные кнопки, нижнее меню

**Хранение данных**
- В браузере: localStorage
- В APK: внутренняя папка приложения + localStorage как резервный слой
- Автосохранение при каждом изменении и при сворачивании приложения
- Экспорт в JSON через диалог «Поделиться» (APK) или скачивание (браузер)
- Импорт из JSON с валидацией структуры и типов

## Использование в браузере

Открыть файл `index.html` в Chrome или другом Chromium-браузере. Данные хранятся в localStorage.

**Установка на главный экран (Android):** разместить файл на HTTPS-хостинге (например [Netlify Drop](https://app.netlify.com/drop)), открыть в Chrome, меню → «Установить приложение».

**Установка на главный экран (iOS):** открыть в Safari, кнопка «Поделиться» → «На экран Домой».

## Сборка APK через Capacitor

### Требования

- Node.js 18+
- Android Studio Hedgehog (2023.1) или новее
- Android SDK API 24+
- ~10 ГБ свободного места на диске

### Шаги

```bash
# Установить зависимости
npm install @capacitor/core @capacitor/cli @capacitor/android
npm install @capacitor/app @capacitor/filesystem @capacitor/share

# Инициализировать проект
npx cap init

# Положить index.html в папку www/
# Добавить Android-платформу
npx cap add android

# Синхронизировать
npx cap sync

# Открыть в Android Studio
npx cap open android
```

В Android Studio: **Build → Build Bundle(s) / APK(s) → Build APK(s)**

Готовый файл: `android/app/build/outputs/apk/debug/app-debug.apk`

Для release-сборки: **Build → Generate Signed App Bundle / APK**

### Версионность

Версия задаётся в `android/app/build.gradle`:

```gradle
defaultConfig {
    versionCode 118  // целое число, растёт с каждым релизом
    versionName "1.18"
}
```

## Структура проекта

```
/
├── www/
│   └── index.html      # всё приложение в одном файле
├── android/            # генерируется Capacitor
├── capacitor.config.json
└── package.json
```

Веб-часть приложения — единственный HTML-файл без внешних зависимостей. Capacitor добавляет только нативную обёртку и плагины файловой системы.
